Task 2

Modify necessary data in the xml file from previous task
  • add additional information - such as, for example, information about the author of the document, date of last modification, etc.
  • there should be about 20 data items (there may be more) - the data should be numbered (numbering in the element, free format - it does not have to be 1, 2, 3, ...)
Then create a cascading CSS style sheet so that you can view the content of the XML document in the browser. Format the information as two sections (using tables, borders, etc.):
  • the first section should contain general information about the document, read from the selected tags (such as the authors of the document, description of its content, etc.);
  • the second section (with columnar or another layout) should contain the correct content from the rest of the tags containing regular data.
When building a CSS document:
  • combine XML document with data with a document describing their formatting (external CSS file);
  • use different formatting layouts (style: display);
  • use different selectors and pseudo-classes;
  • try formatting in the form of a numbered list - showing the number of consecutive nodes/data;
  • use modifications to the background of tables, bullets in lists, etc .;
  • adjust the fonts, colors, and text layout on the page;
  • do the project professionally and neatly, in a way that is attractive to the eye.
When building XML Schema, pay attention to:
  • namespace declarations (standard and custom);
  • precise type determination (built-in: fixed and floating point numbers, dates, currencies, etc.);
  • applying constraints (e.g. string lengths, ranges of numbers), patterns (xsd: pattern), enumerations (xsd: enumeration);
  • the ability to define your own data types (xsd: simpleType, xsd: complexType), use of various content models (e.g. xsd: choice, xsd: sequence);
  • specifying optional and required elements and attributes;
  • specifying attributes for items (xsd: attribute, xsd: attributeGroup);
  • specifying a group of elements (xsd: group);
  • placing links to various places in the document, using keys and selectors (xsd: key, xsd: keyref);
